Abstract

LARGE LANGUAGE MODELS (LLMs) are increasingly used for software development, but their reliability in generating secure code remains a critical concern. SecureCoder addresses these risks by introducing a comprehensive pipeline for security-aware code generation and evaluation.

Our approach goes beyond generation by integrating real-time security feedback directly into the developer's workflow via a JetBrains IDE plugin. Furthermore, we provide a unified evaluation framework capable of rigorously testing models against benchmarks like CWEval, SecCodePLT, and CyberSecEval. By combining advanced techniques such as Graph Neural Networks (GNNs) and symbolic execution, SecureCoder aims to bridge the gap between AI-driven code synthesis and robust software security.

JetBrains IDE Plugin

The SecureCoder JetBrains plugin brings security awareness directly into your IDE. It analyzes code suggestions in real-time, highlighting potential vulnerabilities and suggesting secure alternatives based on our security-first prompting and constraint decoding techniques.

Benchmark CLI Tool

Evaluating the security of code generation models requires rigorous testing. Our Benchmark CLI Tool provides a unified interface to benchmark models from OpenRouter or local agents against state-of-the-art security datasets.

Key Features

  • Multi-Model Support: Benchmark any model available on OpenRouter or running locally.
  • Unified Evaluation: Run benchmarks like CWEval, SecCodePLT, and CyberSecEval with a single command.
  • Automated Reporting: Generate detailed reports on model vulnerability rates.

Powering Techniques

Graph Neural Networks

Integrates GNNs to model code structure at the token level, improving the accuracy and safety of generated code.

Symbolic Execution

Combines symbolic execution with fine-tuning to identify potential vulnerabilities during the model training process.

Obfuscation Grounding

Enhances model resilience by incorporating obfuscation techniques into pre-training to reduce the risk of generating insecure code.
